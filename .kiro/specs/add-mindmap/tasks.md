# Implementation Plan

- [x] 1. マインドマップデータモデルとバリデーション基盤の構築
- [x] 1.1 Go型定義とバリデーションロジックの実装
  - Mindmap構造体（ID、タイトル、ノードJSON、公開ステータス、著者ID、タイムスタンプ）を定義する
  - MindmapNode構造体（ID、テキスト、色、リンクURL、ノート、子ノード配列）を定義する
  - CreateMindmapRequest / UpdateMindmapRequest の入力バリデーションを実装する
  - バリデーションルール: タイトル必須、ノードJSONシリアライズサイズ ≤ 350KB、ノード数 ≤ 500、ノートテキスト ≤ 1000文字
  - ListMindmapsResponse（items、count、nextToken）を定義する
  - ユニットテストでバリデーション境界値（サイズ超過、ノード数超過、テキスト長超過）を検証する
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5, 4.6_

- [x] 1.2 (P) DynamoDB Mindmapsテーブルの Terraform定義
  - Mindmapsテーブル（パーティションキー: id、PAY_PER_REQUEST）を database モジュールに追加する
  - PublishStatusIndex GSI（パーティションキー: publishStatus、ソートキー: createdAt、Projection ALL）を定義する
  - PITR、SSE暗号化、削除保護（本番のみ）を設定する
  - テーブル名をモジュール変数で外部から設定可能にする
  - 出力としてテーブル名とARNをエクスポートする
  - _Requirements: 4.1, 4.3, 9.5_

- [x] 2. マインドマップ管理用Lambda関数の実装
- [x] 2.1 マインドマップ作成Lambda関数の実装
  - Cognito認証からauthorIDを抽出し、リクエストボディをパース・バリデーションする
  - UUIDを生成し、タイムスタンプを付与してDynamoDBにPutItemする
  - 公開ステータスがpublishedの場合、SSG再ビルドをトリガーする
  - 201ステータスで作成されたマインドマップを返却する
  - バリデーションエラー（400）、認証エラー（401）、サーバーエラー（500）のハンドリングを実装する
  - ユニットテストで正常系・バリデーションエラー・認証エラーを検証する
  - _Requirements: 1.1, 1.6, 8.1, 8.9_

- [x] 2.2 (P) マインドマップ取得・一覧Lambda関数の実装
  - 指定IDでDynamoDBからGetItemし、見つからない場合は404を返却する
  - 一覧取得はScanでページネーション（limit、nextToken）対応する
  - Cognito認証を必須とし、全ステータスのマインドマップを返却する
  - ユニットテストで正常系・404・ページネーションを検証する
  - _Requirements: 1.2, 1.3, 1.6, 8.2, 8.3, 8.8_

- [x] 2.3 (P) マインドマップ更新Lambda関数の実装
  - 指定IDのマインドマップを取得し、存在しなければ404を返却する
  - リクエストボディのバリデーション（サイズ、ノード数）を実行する
  - 更新日時を記録し、DynamoDBのUpdateItemで更新する
  - 公開ステータスがdraft→publishedに変更された場合、公開日時を記録しSSG再ビルドをトリガーする
  - ユニットテストで正常系・バリデーションエラー・404・公開切替を検証する
  - _Requirements: 1.4, 1.6, 4.5, 4.6, 8.4, 8.8, 8.9_

- [x] 2.4 (P) マインドマップ削除Lambda関数の実装
  - 指定IDのマインドマップを取得し、存在しなければ404を返却する
  - DynamoDBからDeleteItemで削除する
  - 削除対象が公開済みだった場合、SSG再ビルドをトリガーする
  - ユニットテストで正常系・404を検証する
  - _Requirements: 1.5, 1.6, 8.5, 8.8_

- [x] 3. (P) 公開マインドマップ一覧・詳細Lambda関数の実装
  - 公開一覧: PublishStatusIndex GSIでpublishStatus=publishedをクエリし、ページネーション対応する
  - 公開詳細: 指定IDでGetItemし、publishStatus=publishedでなければ404を返却する
  - 認証不要（Cognito Authorizer未設定）で動作する
  - ユニットテストで正常系・非公開マインドマップの404・ページネーションを検証する
  - _Requirements: 3.1, 8.6, 8.7, 8.8_

- [x] 4. API GatewayルートとLambda Terraform定義の構築
- [x] 4.1 マインドマップLambda関数のTerraform定義
  - 7つのLambda関数（create、get、list、update、delete、get_public、list_public）をlambdaモジュールに追加する
  - 環境変数にMINDMAPS_TABLE_NAME（新規）を追加し、既存の環境変数（BUCKET_NAME等）も設定する
  - ARM64アーキテクチャ、provided.al2023ランタイムで定義する
  - IAMポリシーにMindmapsテーブルへのアクセス権限を付与する
  - Makefileにmindmaps用ビルドターゲットを追加する
  - _Requirements: 9.4, 9.5_

- [x] 4.2 API Gatewayルートの定義
  - `/admin/mindmaps`リソースとCRUDメソッド（POST、GET）を定義する（Cognito Authorizer付き）
  - `/admin/mindmaps/{id}`リソースとメソッド（GET、PUT、DELETE）を定義する（Cognito Authorizer付き）
  - `/public/mindmaps`リソースとメソッド（GET）を定義する（認証なし）
  - `/public/mindmaps/{id}`リソースとメソッド（GET）を定義する（認証なし）
  - 各メソッドにLambda統合とCORS OPTIONSメソッドを設定する
  - 環境別設定ファイル（dev/prd）にMindmapsテーブル名変数を追加する
  - _Requirements: 8.1, 8.2, 8.3, 8.4, 8.5, 8.6, 8.7, 9.5_
